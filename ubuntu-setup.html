Know Your System

Before following any commands in this guide, run these to find your specific values. Replace username and YOUR_WIFI_NAME throughout the document with what you find here.

Your Username

This appears in file paths like /home/username/ throughout the guide.

whoami

Or confirm the full home directory path:

echo $HOME

Your Wi-Fi Network Name

Needed when running nmcli connection modify commands in the DNS section.

nmcli connection show --active

Look for the entry with wifi in the TYPE column — the name in the first column is your Wi-Fi network name.

Your Hardware

Laptop model:

sudo dmidecode -s system-product-name

Processor:

cat /proc/cpuinfo | grep "model name" | head -1

Full system info (opens GUI):

gnome-control-center info-overview

Your Ubuntu Version

Some commands and settings vary between Ubuntu releases — check yours first.

lsb_release -a

This guide was written for Ubuntu 26.04 LTS. Most commands also apply to 24.04 LTS.

Your DNS and Network Status

Useful baseline to run before making any DNS changes so you know your starting state.

resolvectl status

Check current active connections:

nmcli connection show --active

Check routing table:

ip route show

Installing & Updating Apps

Package Managers Overview

  • apt — main package manager for most apps
  • snap — Ubuntu's sandboxed app system (auto-updates)
  • flatpak — cross-distro app system (manual updates needed)
  • .deb files — manually downloaded packages

apt Commands

Update package list and upgrade all apt apps:

sudo apt update && sudo apt upgrade

Install a specific app:

sudo apt install <appname>

Fix interrupted installs:

sudo dpkg --configure -a
sudo apt install -f

Install common media codecs:

sudo apt install ubuntu-restricted-extras

Snap Commands

Manually update all snaps:

sudo snap refresh

Remove a snap:

sudo snap remove <appname>

Flatpak Commands

Add Flathub repository (first-time setup):

sudo apt install flatpak
sudo flatpak remote-add --if-not-exists flathub https://dl.flathub.org/repo/flathub.flatpakrepo

Update all Flatpak apps:

flatpak update

Uninstall a Flatpak app:

flatpak uninstall <app.id>

Installing .deb Files

cd ~/Downloads
sudo dpkg -i filename.deb
sudo apt -f install    # fixes any missing dependencies

Update Everything at Once

sudo apt update && sudo apt upgrade && sudo snap refresh && flatpak update

Automated Updates (Anacron)

System updates are handled automatically via anacron — see Section 6 for the full anacron setup. The monthly update job runs:

apt-get update && apt-get upgrade -y && snap refresh && flatpak update -y

Run updates manually anytime:

sudo apt update && sudo apt upgrade && sudo snap refresh && flatpak update

View current anacron jobs:

sudo cat /etc/anacrontab

Firewall Setup (UFW)

Enable the firewall:

sudo ufw enable

Check firewall status:

sudo ufw status

Allow a specific port:

sudo ufw allow 22    # example: SSH

Deny a specific port:

sudo ufw deny 23

Disable the firewall (if needed):

sudo ufw disable
Default UFW rules are sensible for a personal laptop — no additional configuration needed beyond enabling it.

Quad9 DNS over HTTPS

Encrypts all DNS lookups system-wide so your ISP cannot see which domains you're querying. Quad9 also blocks known malicious domains automatically.

Step 1: Configure systemd-resolved

sudo nano /etc/systemd/resolved.conf

Edit these lines (remove # and set values):

DNS=9.9.9.9#dns.quad9.net
FallbackDNS=149.112.112.112#dns.quad9.net
DNSOverTLS=opportunistic

Save and restart:

sudo systemctl restart systemd-resolved
Use opportunistic rather than yes — strict mode prevents ProtonVPN from taking over DNS when the VPN connects, breaking browsing. Opportunistic uses TLS when available but hands off gracefully to ProtonVPN's DNS when the VPN tunnel is active. Privacy is identical since WireGuard/OpenVPN already encrypts all traffic including DNS.

Step 2: Configure NetworkManager

sudo nano /etc/NetworkManager/NetworkManager.conf

Add under the [main] section:

dns=systemd-resolved

Restart NetworkManager:

sudo systemctl restart NetworkManager

Step 3: Set Per-Connection DNS

Find your Wi-Fi connection name:

nmcli connection show

Apply Quad9 to your Wi-Fi connection:

nmcli connection modify "YOUR_WIFI_NAME" ipv4.dns "9.9.9.9 149.112.112.112" ipv4.ignore-auto-dns yes
nmcli connection down "YOUR_WIFI_NAME" && nmcli connection up "YOUR_WIFI_NAME"
Repeat this step for each new Wi-Fi network you connect to — DNS settings are stored per connection.

Verify DNS is Working

resolvectl status

Look for: Current DNS Server: 9.9.9.9#dns.quad9.net and +DNSOverTLS on both Global and your Wi-Fi link.

Firefox DoH Settings

  • Settings → Privacy & Security → DNS over HTTPS
  • Set to Max Protection
  • Provider: Custom → https://dns.quad9.net/dns-query

Custom System Changes

Startup Applications

These apps are configured to launch automatically on every login — keeping your privacy and usability setup active without manual intervention.

Caffeine Indicator

Add via GNOME Tweaks → Startup Applications → click + and add Caffeine Indicator. Or via terminal:

mkdir -p ~/.config/autostart
cp /usr/share/applications/caffeine-indicator.desktop ~/.config/autostart/

Verify it's set to autostart:

ls ~/.config/autostart/

The coffee cup icon will appear in the top bar automatically on every login — click to activate when casting, click again when done.

ProtonVPN

Configure within the ProtonVPN app rather than GNOME Tweaks — go to ProtonVPN → Settings → General and set:

  • Auto connect: US for fastest US server, or Fastest for fastest server regardless of country
  • Start app minimized: on — launches silently to the tray without opening the full window

Then add ProtonVPN to startup applications via GNOME Tweaks → Startup Applications → click + and add ProtonVPN.

Make sure Kill Switch remains off in ProtonVPN settings — as documented in Section 7, Kill Switch caused routing conflicts on Ubuntu that broke browsing.

Chromecast

Google Cast works on Linux through Chromium-based browsers. Brave (already installed) supports casting natively — no additional software needed.

  • Click the three-dot menu in Brave → Cast... or Save and share → Cast...
  • Your Chromecast device should appear if it's on the same Wi-Fi network
  • Make sure your laptop and Chromecast are on the same Wi-Fi network — casting won't work across different networks

Caffeine — Prevent Sleep During Casting

Prevents the computer from sleeping or disconnecting during Chromecast sessions. Adds a coffee cup icon to the system tray to toggle on/off.

sudo apt install caffeine

Launch Caffeine Indicator from your app menu. Click the coffee cup icon in the top bar to activate — the cup appears full when active.

Remember to turn Caffeine off when done casting — leaving it active permanently defeats battery optimization settings and drains the battery faster when unplugged. If the lock screen still activates while casting, go to Settings → Privacy → Screen Lock → increase the Automatic Screen Lock Delay.

Bluetooth Default Off on Boot

By default Ubuntu restores the previous bluetooth state on reboot. This disables it by default on every boot — reducing attack surface and saving battery. You can still enable it manually whenever needed via Settings → Bluetooth.

sudo nano /etc/bluetooth/main.conf

Find and change:

AutoEnable=false

Save, then restart the bluetooth service:

sudo systemctl restart bluetooth

Volume Control

PulseAudio Volume Control gives per-app volume control and surfaces additional volume headroom not available in the standard Ubuntu slider — without distortion risk.

sudo apt install pavucontrol

Launch from your app menu. Adjust per-app and system-wide volumes independently.

If you still need volume above 100% after using pavucontrol, you can enable over-amplification — but avoid leaving it maxed on built-in speakers for extended periods: gsettings set org.gnome.desktop.sound allow-volume-above-100-percent true

GNOME Tweaks

GNOME Tweaks unlocks additional customization options not available in the standard Settings app — fonts, window behavior, startup apps, and more.

Install GNOME Tweaks:

sudo apt install gnome-tweaks

Launch from your app menu — useful settings to explore include window titlebars, font sizes, and startup application management.

Click to Minimize

Makes clicking an already-open app's dock icon minimize it, rather than just switching to it — more consistent with Windows behavior.

gsettings set org.gnome.shell.extensions.dash-to-dock click-action 'minimize'

Takes effect immediately — no reboot needed.

Notifications

  • Disable lock screen notifications: Settings → Notifications → toggle off Lock Screen Notifications
  • Disable per-app notifications: Settings → Notifications → select individual apps
  • Do Not Disturb: click time/date in top bar → toggle Do Not Disturb

Disable screenshot shutter sound:

gsettings set org.gnome.desktop.sound event-sounds false

Clock Format (12-hour)

gsettings set org.gnome.desktop.interface clock-format '12h'

Touchpad Scroll Speed

  • Settings → Mouse & Touchpad → Touchpad → Scrolling Speed slider

Battery Optimization

Install and enable TLP:

sudo apt install tlp tlp-rdw
sudo systemctl enable tlp
sudo systemctl start tlp

Install PowerTop:

sudo apt install powertop
sudo powertop --auto-tune
  • Settings → Power → set Power Mode to Power Saver when unplugged
  • Enable Automatic Suspend on battery

Set Default PDF Reader (Okular)

xdg-mime default org.kde.okular.desktop application/pdf

Automatic Security Updates

sudo apt install unattended-upgrades
sudo dpkg-reconfigure unattended-upgrades

Disable Crash Reports

sudo systemctl disable apport

Kernel Security Hardening

sudo nano /etc/sysctl.conf

# Add at bottom:
net.ipv4.ip_forward=0
net.ipv4.icmp_echo_ignore_broadcasts=1
net.ipv4.conf.all.accept_source_route=0
net.ipv4.tcp_syncookies=1

Apply changes:

sudo sysctl -p

Metadata Stripping (before sharing files)

mat2 filename.pdf

Secure File Deletion

srm -r ~/path/to/sensitive/file

Security Scanning & Automation

Anacron — Automated Scheduling

Anacron is used instead of cron for laptop use — it runs missed jobs the next time the laptop is on rather than skipping them entirely if the laptop was off or asleep at the scheduled time.

Install Anacron:

sudo apt install anacron

Edit the anacron schedule:

sudo nano /etc/anacrontab

Add these lines at the bottom:

7	10	clamav-scan	freshclam && clamscan -r -i --log=/home/username/clamav-scan.log /home/username/
7	15	rkhunter-scan	rkhunter --check --skip-keypress --report-warnings-only --logfile /home/username/rkhunter-scan.log
7	20	chkrootkit-scan	chkrootkit 2>&1 | grep -i "infected\|warning" >> /home/username/chkrootkit-scan.log
30	25	system-update	apt-get update && apt-get upgrade -y && snap refresh && flatpak update -y

The columns mean: period in days | delay in minutes after boot | job name | command

Replace username with your actual username throughout. Jobs are staggered by delay to avoid running simultaneously.

Remove AC Power Requirement

By default anacron only runs when plugged in. To allow it to run on battery:

sudo mkdir -p /etc/systemd/system/anacron.service.d/
sudo nano /etc/systemd/system/anacron.service.d/override.conf

Add this content:

[Unit]
ConditionACPower=

Save, then reload:

sudo systemctl daemon-reload
sudo systemctl restart anacron

Verify Anacron is Running

sudo systemctl status anacron

Look for active (running) and all four jobs listed as scheduled. The override.conf drop-in should be visible confirming the AC power fix is applied.

Purpose of Each Scan

ClamAV

  • Scans files for known malware, viruses, and trojans
  • Works from a database of known threat signatures updated regularly via freshclam
  • Best at catching malicious files downloaded from the internet or received as attachments
  • Think of it as a file-level inspector

rkhunter

  • Scans for rootkits — malware that hides itself deep in the system to avoid detection
  • Checks system binaries against a known-good baseline set when you ran --propupd
  • Also checks for suspicious network ports, hidden files, and known attack signatures
  • Think of it as a system integrity checker

chkrootkit

  • Also scans for rootkits using different detection methods than rkhunter
  • Running both gives broader coverage since they use independent approaches
  • More prone to false positives on Ubuntu than rkhunter
  • Think of it as a second opinion on system integrity

Why run all three?

  • ClamAV catches things rkhunter/chkrootkit miss (downloaded malware)
  • rkhunter and chkrootkit catch things ClamAV misses (deep system compromise)
  • No single scanner catches everything — layered scanning gives the most complete picture

Monthly Log Review — One Command

Run this once a month for a quick summary of all three scan results:

echo "=== ClamAV ===" && grep "Infected files" /home/username/clamav-scan.log | tail -3
echo "=== rkhunter ===" && grep "Warning" /home/username/rkhunter-scan.log | tail -5
echo "=== chkrootkit ===" && grep "INFECTED" /home/username/chkrootkit-scan.log | tail -5

A clean result looks like:

  • ClamAV: Infected files: 0
  • rkhunter: no output, or only warnings you've already identified as false positives
  • chkrootkit: no output, or known false positives on basic utilities

Interpreting Results & What to Do

ClamAV finds an infected file:

  • Note the file path reported
  • Do not open or run the file
  • Search the virus name online to understand what it is
  • If it's in Downloads, delete it immediately:
rm ~/Downloads/suspicious-filename

rkhunter or chkrootkit warnings:

  • Check if the warning existed in your previous scan — recurring warnings are usually false positives
  • Verify flagged system files belong to legitimate packages:
sudo dpkg -S /path/to/flagged/file

If dpkg returns a package name it's legitimate. If it returns nothing, investigate further.

Check if a warning matches a recent system update:

grep "install\|upgrade" /var/log/dpkg.log | tail -20

Escalation Guide

  • False positive (most common) — verify with dpkg, search online, confirm it appeared in previous scans. No action needed
  • Suspicious file in Downloads — delete the file, run a full ClamAV scan, change passwords if the file was opened
  • Unexpected system file change — check dpkg.log for recent updates; if it matches a legitimate update, no action needed
  • Genuine infection suspected — disconnect from internet, don't log into any accounts from this machine, restore from a clean Timeshift snapshot, change passwords from a different device
On a well-maintained Ubuntu system with a privacy-focused setup, the overwhelming majority of scan results will be clean or known false positives. Genuine infections are rare — the scans are insurance, not a daily concern.

ClamAV (Malware) — Manual Use

Update virus definitions:

sudo freshclam

Scan home folder manually:

clamscan -r --bell -i ~/

Scan a specific file before opening:

clamscan ~/Downloads/filename.pdf

rkhunter (Rootkits) — Manual Use

Update and set system baseline (run once after install):

sudo rkhunter --update
sudo rkhunter --propupd

Run a manual scan:

sudo rkhunter --check --skip-keypress

View warnings:

sudo cat /var/log/rkhunter.log | grep -i "error\|fail\|warn"
rkhunter 1.4.6 has known update mirror issues on Ubuntu. If --update fails, this is expected — the existing definitions still work for rootkit detection.

chkrootkit (Rootkits) — Manual Use

Run a manual scan:

sudo chkrootkit
Many chkrootkit warnings on basic utilities (basename, echo, date) are false positives on Ubuntu. Verify with: sudo dpkg -S /bin/<command>

Timeshift — System Snapshots

Timeshift takes snapshots of your system that you can roll back to if something goes wrong — a broken update, misconfigured system file, or failed install. It protects your system configuration but not personal files in /home — keep a separate backup for personal data.

Install Timeshift:

sudo apt install timeshift

Open the setup wizard:

Launch Timeshift from your app menu — it runs a setup wizard on first open:

  • Snapshot type: RSYNC (recommended)
  • Snapshot location: main drive (external drive is better if available)
  • Schedule: Weekly — keep 3; Monthly — keep 2; Daily optional
  • User home directories: leave excluded (not designed for personal file backup)
Timeshift manages its own scheduling and handles missed snapshots automatically — no cron or anacron needed.

Take a Manual Snapshot

Always take one before major system changes, updates, or installing unfamiliar software:

sudo timeshift --create --comments "Before major change"

Or via the GUI: open Timeshift → click Create.

Restore from a Snapshot

  • Open Timeshift → select the snapshot → click Restore → reboot
  • If the system won't boot at all, Timeshift can be accessed from a live Ubuntu USB

Useful Timeshift Commands

List available snapshots:

sudo timeshift --list

Delete old snapshots:

sudo timeshift --delete

Brave — PWA Host (Claude & Google Messages)

Brave is used only as a PWA host. It has known bugs on Linux where PWAs don't appear in the app menu or dock automatically — the workaround is to manually create a .desktop file for each PWA.

Step 1: Install the PWA in Brave

  • Go to the site (e.g. claude.ai) and sign in
  • Click the three-dot menu → look for "Install Claude..." or "Install as app"
  • Confirm the install — Brave will open it in a standalone window
  • If no install option appears, check brave://apps to see installed PWAs

Step 2: Create a .desktop File Manually

For Claude:

nano ~/.local/share/applications/claude-pwa.desktop
[Desktop Entry]
Name=Claude
Exec=/usr/bin/brave-browser --app=https://claude.ai
Icon=brave-browser
Type=Application
Categories=Network;

For Google Messages:

nano ~/.local/share/applications/messages-pwa.desktop
[Desktop Entry]
Name=Google Messages
Exec=/usr/bin/brave-browser --app=https://messages.google.com/web
Icon=brave-browser
Type=Application
Categories=Network;

Step 3: Refresh the App Menu

update-desktop-database ~/.local/share/applications/

Apps should now appear in the GNOME app menu and can be pinned to the dock by right-clicking → Pin to Dash.

Changing a PWA Icon

Brave auto-generates .desktop files with random names rather than readable ones. To find and edit them:

  • Open the Files app and press Ctrl+H to show hidden folders
  • Navigate to Home → .local → share → applications
  • Right-click each brave- file → Open With Text Editor
  • Check the Name= line to identify which file is Claude vs Google Messages
  • Add or update the Icon= line with the full path to your PNG image

For example, if you saved your icon to a folder in Pictures:

Icon=/home/username/Pictures/icons/claude.png

Save the file — the icon should update immediately. If not, run:

update-desktop-database ~/.local/share/applications/

And log out/in to force the dock icon to refresh.

If Brave overwrites your icon after an update, just repeat the process — the .desktop files are easy to find and edit via the Files app.

Finding PWA .desktop Files

Brave generates random filenames for PWA desktop entries. The easiest way to find them is via the Files app (Ctrl+H → .local → share → applications). Alternatively via terminal:

ls ~/.local/share/applications/

Look for files starting with brave- — open each with a text editor to identify which is Claude vs Google Messages by checking the Name= line.

Brave Privacy Settings

Go to brave://settings/privacy:

  • Shields — keep enabled
  • Fingerprinting protection — set to Strict
  • Send a "Do Not Track" request — enable
  • Block third-party cookies — enable
  • WebRTC IP handling — set to Disable non-proxied UDP — critical with ProtonVPN to prevent real IP leaking through WebRTC
  • Usage ping / diagnostics — disable all data sharing

Go to brave://settings/shields:

  • Trackers & ads blocking: Aggressive
  • Upgrade connections to HTTPS: Strict
  • Block fingerprinting: Strict, may break sites
  • Block cookies: Only cross-site cookies

Disable Unused Brave Features

These features are off by default in current Brave versions but worth verifying:

  • Brave Rewards / BAT → brave://rewards — disable
  • Brave Wallet → brave://settings/wallet → set to None
  • Brave News → toggle off at bottom of new tab page or brave://settings/newTab
  • Brave VPN — Linux only, may not appear; use ProtonVPN instead

DNS over HTTPS in Brave

Go to brave://settings/security:

  • Under Use secure DNS → select With Custom
  • Enter: https://dns.quad9.net/dns-query
When ProtonVPN is active it overrides this — that's expected and fine.

Extensions for Brave

Keep extensions minimal in Brave — it's only used for PWAs.

Worth Installing

  • uBlock Origin Lite — use Lite version in Brave (Chromium's Manifest V3 restrictions prevent the full version); complements Brave Shields
  • Proton Pass — consistent password manager

Skip in Brave

  • Privacy Badger — redundant with Shields
  • ClearURLs — Shields handles tracking params
  • Facebook Container — Firefox-only
  • Any VPN extension — use system ProtonVPN

ProtonVPN Protocol & Compatibility

  • Use WireGuard protocol (default) — fastest, most modern, best battery life
  • If browsing breaks when VPN connects, switch to OpenVPN (UDP) as fallback
  • PWAs staying connected with VPN requires DNSOverTLS=opportunistic in systemd-resolved — see Section 4
  • If kill switch is enabled and VPN drops, delete stale connections with: nmcli connection delete pvpn-killswitch

PWA-Specific Considerations

  • PWAs share Brave's cookie jar — signing out of one Google service may affect Google Messages
  • If a PWA breaks, try temporarily lowering Shields via the lion icon in the address bar
  • Google Messages requires your Pixel to be online — it routes through your phone
  • Re-pair Google Messages if the session expires: Pixel Messages app → menu → Device Pairing

General Maintenance

Regular maintenance keeps your laptop running well for years. Split into physical cleaning, hardware health monitoring, and software hygiene.

Physical Cleaning

Never spray anything directly onto the laptop — always apply liquid to a cloth first. Avoid household cleaners (Windex, multi-surface sprays) on any part of the laptop.

Screen

Most sensitive component — the anti-glare coating can be permanently damaged by alcohol above 70% or abrasive materials.

  • For dust: dry microfiber cloth only
  • For smudges: microfiber cloth slightly dampened with distilled water only
  • Never use: Windex, glass cleaners, paper towels, or alcohol wipes

Keyboard & Chassis

More durable than the screen but still requires care.

  • For cleaning and sanitizing: 70% isopropyl alcohol wipes — safe for keyboard and chassis surfaces (Dell recommended)
  • For debris under keys: compressed air
  • Never spray liquid near ports or vents

Vents & Ports

  • Compressed air only — never use liquid near vents or ports
  • Clean vents every 6-12 months — dust buildup causes thermal throttling and reduced performance
  • Hold the fan still while blowing air into vents to avoid overspinning it

Recommended Cleaning Kit

  • Microfiber cloths (pack of several)
  • Distilled water — for screen only
  • 70% isopropyl alcohol wipes — for keyboard and chassis
  • Compressed air can — for vents and keyboard debris

Battery Health

Check battery health:

upower -i /org/freedesktop/UPower/devices/battery_BAT0

Key values to monitor:

  • energy-full vs energy-full-design — compare these to calculate remaining capacity
  • capacity % — below 80% warrants a replacement
  • charge-cycles — if available, high cycle counts indicate aging battery
Run this check every few months. If capacity drops below 80% within a year of a new battery installation, claim the warranty.

Battery Charging Settings

  • Go to Settings → Power → Battery Charging
  • Preserve Battery Health — caps charge at ~80%, better for long-term battery longevity. Best if you mostly work plugged in
  • Maximize Charge — charges to 100%, better if you regularly need maximum runtime away from power

Battery Replacement

  • Find your battery part number via your service tag at dell.com/support
  • Buy genuine Dell replacement parts directly from Dell for lithium-ion batteries — non-genuine batteries are a safety risk
  • Consider having a repair shop handle installation — buying the part yourself and paying labor only saves $30-50 vs shop-supplied parts
  • Battery replacement has zero impact on Ubuntu or any software — the SSD is completely separate

SSD Health

Install smartmontools (one time):

sudo apt install smartmontools

Run SSD health check:

sudo smartctl -a /dev/nvme0n1

Key values to monitor:

  • SMART overall-health: should say PASSED
  • Available Spare: should stay well above the threshold (50%)
  • Percentage Used: manufacturer's estimate of drive life consumed — below 50% is healthy
  • Media and Data Integrity Errors: should be 0
  • Temperature: normal operating range is 25-50°C
Run this check every 6 months. If Available Spare drops below the threshold or errors appear, back up immediately.

RAM Usage

Check current memory usage:

free -h

Look at the available column — if it's consistently low (under 1-2GB) your system is under memory pressure and performance may suffer.

Software Hygiene

Clean up unused packages:

sudo apt autoremove && sudo apt autoclean

Check disk usage:

df -h

Look at the Use% column for your main drive — above 85% is worth addressing by clearing unused files or apps.

Check what's taking up space:

du -sh ~/* | sort -hr | head -20

Timeshift Snapshots

  • Open Timeshift from your app menu to verify scheduled snapshots are running
  • Take a manual snapshot before any major system changes or updates
  • Keep 3-5 snapshots — older ones are deleted automatically based on your schedule setting

Thermal Paste (Long Term)

After 4-5 years the thermal paste between the CPU and heatsink can dry out, causing higher temperatures and the fan running constantly. Signs of degradation:

  • Fan running loudly under light workloads
  • System feeling hot to the touch during normal use
  • Performance noticeably slower than it used to be

This is a repair shop job — ask them to reapply thermal paste when you bring the laptop in for battery replacement, as they'll already have the laptop open.
